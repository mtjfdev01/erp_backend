Bank Alfalah APG — credit/debit card only (ddr_server)
======================================================

SETUP
-----
1. ALFALAH_ENV=sandbox | production
   Required: APG_MERCHANT_ID, APG_STORE_ID, APG_MERCHANT_HASH, USERNAME, PASSWORD
   Production: APG_KEY1, APG_KEY2 (16 chars each)
2. API_PUBLIC_BASE_URL=https://your-api.example.com
   APG_HS_RETURN_URL=https://your-api.example.com/donations/public/alfalah/return
   APG_RETURN_URL=https://your-api.example.com/donations/public/alfalah/return
   (base URLs only — server appends ?donationId={id} per payment)
   BASE_Frontend_URL=https://www.mtjfoundation.org
3. APG merchant portal (whitelist with bank):
   Return URL = APG_RETURN_URL base above
   Listener URL = {API}/donations/public/alfalah/listener
   Card channel = 1001 (page redirection)

FLOW (APG guide)
----------------
Step 1 — Server POST to {APG}/HS/HS/HS (HS_IsRedirectionRequest=1) → AuthToken.
Step 2 — API returns SSO form; donor browser POSTs to {APG}/SSO/SSO/SSO.
Fallback — If server handshake fails, API returns HS/HS/HS form (cardStep 1) for browser.
Step 2b — If browser used step 1, APG redirects to HS_ReturnURL; API auto-POSTs SSO HTML.
Step 3 — Donor pays on APG secure card page.
Step 4 — APG redirects to ReturnURL with O=order id; API calls IPN OrderStatus and redirects frontend.

CREATE DONATION
---------------
POST /donations (or public create route)
  donation_method: "alfalah"

Response:
{
  "donationId": 123,
  "transactionReference": "123",
  "cardStep": 1,
  "formAction": "https://payments.bankalfalah.com/HS/HS/HS",
  "formFields": { "HS_ChannelId": "1001", ... }
}

Frontend: postGatewayForm(formAction, formFields) once.

PUBLIC ROUTES
-------------
GET  /donations/public/alfalah/return   — Step 2 SSO HTML + post-payment IPN sync
POST /donations/public/alfalah/listener — IPN webhook (?url=... then GET that URL)
GET  /donations/public/alfalah/status   — credentials configured check
GET  /donations/:id/provider-status     — polls IPN when donation_method is alfalah
